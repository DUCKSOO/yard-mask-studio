# yard-mask-studio 사용자 가이드 v1.0

**대상**: 라벨링 작업자·검수자. 서버 설치나 코드 수정 없이 브라우저만으로 작업하는 경우를 기준으로 한다.

> 라벨 판정 기준(occupied/non_occupied/ignore)은 [labeling_guide.md](./labeling_guide.md) 를 참고한다.  
> 서버 설치·환경 구성은 [README](../README.md) 를 참고한다.

---

## 1. 시작 전 확인사항

1. 팀에서 안내한 **브라우저 URL**로 접속한다 (예: `http://localhost:5173`).
2. 서버(백엔드)가 켜져 있어야 한다. 흰 화면이나 API 오류가 뜨면 담당자에게 문의한다.
3. **tenant ID**와 **dataset ID**를 미리 확인한다. 담당자가 사전에 공유한다.

---

## 2. 화면 구성 (탭)

상단 **데이터셋 · 라벨링 · 검수 · Export** 탭으로 기능이 나뉩니다. 오른쪽 **작업 대상**에는 현재 선택된 `tenant / dataset`이 표시됩니다.

| 탭 | 용도 |
|----|------|
| **데이터셋** | tenant 입력, 목록 조회, 새 데이터셋 생성, 타일 생성, 행의 **▶ 라벨링**으로 라벨 화면 이동 |
| **라벨링** | 타일·클래스·브러시·SAM·저장 (캔버스 + 좁은 사이드바) |
| **검수** | 검수 큐 전체 화면 |
| **Export** | U-Net ZIP 내보내기 + 설정 영향도 |

```
┌────────────────────────────────────────────────────────┐
│  yard-mask-studio                                      │
│  [데이터셋][라벨링][검수][Export]   작업 대상 t / ds   │
├────────────────────────────────────────────────────────┤
│  (선택한 탭에 따라 전체 영역이 바뀜)                     │
└────────────────────────────────────────────────────────┘
```

**라벨링** 탭 상세:

```
┌──────────────────────┬─────────────────────────────────┐
│  사이드바 (왼쪽)      │  캔버스 영역 (오른쪽)              │
│  · 타일 새로고침      │  타일 이미지 + 마스크 오버레이      │
│  · 진행률 바          │  그리드 오버레이                   │
│  · TileNavigator      │                                  │
│  · ClassPanel         │                                  │
│  · ToolBar + 슬라이더 │                                  │
│  · 액션 버튼          │                                  │
└──────────────────────┴─────────────────────────────────┘
```

## 3. 탭·기능별 설명

### 3-1. 데이터셋 탭

- **tenant** 입력 후 **목록 새로고침**으로 등록된 데이터셋을 표에서 확인합니다.
- 행을 클릭하면 해당 데이터셋이 **작업 대상**으로 선택됩니다.
- **▶ 라벨링**으로 곧바로 라벨링 탭으로 이동합니다.
- **선택된 데이터셋 · 타일 생성**에서 GeoTIFF(파일명)를 두고 **타일 생성**을 실행할 수 있습니다.
- **새 데이터셋 생성** 토글에서 ID·GeoTIFF를 넣고 **생성** 또는 **생성+타일**을 사용합니다.

### 3-2. 라벨링 탭

- 작업 대상은 데이터셋 탭에서 선택합니다. 미선택 시 안내 문구가 보입니다.
- **타일 목록 새로고침** 후 TileNavigator에서 타일을 고릅니다.

### 3-3. 검수 탭

- 테넌트 단위 검수 큐입니다. 필터·승인·거부는 기존과 동일합니다.

### 3-4. Export 탭

- **작업 대상** 데이터셋에 대해 U-Net Export를 실행합니다.
- 설정 영향도 분석 위젯도 이 탭에 있습니다.

## 4. 라벨링 사이드바 (세부)

### 4-1. 진행률 바

- 새로고침 직후 `전체 N | labeled M | approved P` 한 줄로 표시된다.
- 파란 막대: labeled 비율, 초록 막대: approved 비율.

### 4-2. TileNavigator (타일 그리드)

- 상단 탭으로 상태별 필터: **전체 / 미라벨 / labeled / approved / rejected**
- 각 버튼은 상태에 따라 색상이 다르다.

| 색상 | 상태 |
|------|------|
| 회색 | unlabeled (미라벨) |
| 파랑 | labeled |
| 초록 | approved |
| 빨강 | rejected |

- 버튼을 클릭하면 해당 타일 이미지와 기존 마스크를 캔버스에 불러온다.
- 타일 ID가 길 경우 뒷부분만 표시되며, 마우스를 올리면 전체 ID가 tooltip으로 보인다.

### 4-3. ClassPanel (클래스 선택)

브러시로 칠할 **클래스(색상)**를 고른다. 현재 프로젝트 기본값:

| 클래스 | ID | 의미 |
|--------|----|------|
| non_occupied | 0 | 야드 바닥·통로 |
| occupied | 1 | 컨테이너·장비·적재물 |
| ignore | 255 | 경계 불확실·가림 |

### 4-4. ToolBar (도구)

| 도구 | 설명 |
|------|------|
| 브러시(B) | 선택한 클래스로 마스크를 칠한다 |
| 지우개(E) | 해당 픽셀을 배경(0)으로 지운다 |
| 점(+) | SAM에 전경 프롬프트 점을 찍는다 |
| 점(−) | SAM에 배경 프롬프트 점을 찍는다 |
| 박스 | SAM에 바운딩 박스 프롬프트를 그린다 |
| 이동(P) | 캔버스를 드래그로 이동한다 |

**반경 슬라이더**: 브러시·지우개 도구가 선택된 경우에만 표시된다 (2 ~ 80px, 기본 10px).

**SAM 실행**: **양성 점(전경)** 또는 **박스**가 있을 때 활성화된다. (음성 점만으로는 실행되지 않는다.)

- **여러 덩어리의 자재**(서로 떨어진 객체)에 양성을 한꺼번에 많이 두면 SAM이 “한 객체”로 해석해 마스크가 비정상적으로 커질 수 있다. **SAM 자동**이 켜져 있으면 양성(좌클릭)마다 프롬프트가 비워지고 곧바로 추론해 **자재 하나씩** 처리하기 쉽다(성공 시 프롬프트 비움). **SAM 자동**을 끄면 양성을 여러 개 쌓은 뒤 **SAM 실행** 한 번으로도, 내부적으로는 **양성마다 따로** 추론한 뒤 마스크를 합친다.
- **같은 자재**를 다듬을 때는 **양성 1점 + 음성 점**만 쌓은 뒤 SAM을 돌린다(이때는 한 번의 추론 요청). **양성이 2개 이상**이면 음성 점은 요청에 넣지 않으며, 화면에 안내 메시지가 붙는다.
- **박스** 프롬프트가 있으면 SAM 요청에는 **박스만** 포함되고 점은 무시된다.

결과가 마음에 들지 않으면 Undo 후 브러시로 직접 수정한다.

### 4-5. 액션 버튼

| 버튼 | 동작 |
|------|------|
| Undo | 직전 마스크 변경 취소 |
| Redo | 취소된 변경 다시 적용 |
| 저장 | 현재 마스크를 서버에 저장 (타일 상태 → `labeled`) |
| 다시 불러오기 | 서버에 저장된 마스크로 초기화 |

## 5. 캔버스 조작

| 조작 | 방법 |
|------|------|
| 확대·축소 | 마우스 **휠** |
| 캔버스 이동 | **휠 버튼** 드래그 또는 이동 도구(P) |
| 마스크 칠하기 | 브러시(B) 선택 후 **클릭·드래그** |
| 마스크 지우기 | 지우개(E) 선택 후 **클릭·드래그** |
| SAM 점 프롬프트 | 점(+)·점(−) 도구 선택 후 클릭 |
| SAM 박스 프롬프트 | 박스 도구 선택 후 드래그 |

**그리드 오버레이**: 야드 구획 기준 격자가 반투명으로 표시된다. 캔버스 확대 시 격자도 같이 확대된다.

## 6. 단축키 요약

| 단축키 | 동작 |
|--------|------|
| **Ctrl+Z** / **Cmd+Z** | Undo |
| **Ctrl+Y** / **Cmd+Y** | Redo |
| **Ctrl+S** / **Cmd+S** | 저장 |
| **B** | 브러시 도구로 전환 |
| **E** | 지우개 도구로 전환 |
| **P** | 이동(팬) 도구로 전환 |

> 텍스트 입력 필드에 포커스가 있을 때는 단축키가 동작하지 않는다.

## 7. 일반적인 작업 순서 (라벨러)

1. **데이터셋** 탭에서 tenant·목록 확인 후 작업할 데이터셋 선택 (또는 생성·타일 생성).
2. **라벨링** 탭으로 이동 (**▶ 라벨링** 또는 상단 탭).
3. **타일 목록 새로고침** 후 TileNavigator에서 **미라벨(회색)** 타일 클릭.
4. ClassPanel에서 칠할 **클래스 선택**.
5. (선택) SAM: **자재마다 양성 1점**으로 후보를 만들고, 필요하면 **음성 점**으로 같은 자재만 다듬은 뒤 SAM을 다시 실행한다. 여러 자재는 양성을 반복해 찍는다. 이후 브러시·지우개로 보정.
6. **Ctrl+S** 또는 저장 버튼.
7. 다음 타일 클릭 → 반복.

---

## 8. 일반적인 작업 순서 (검수자)

1. **검수** 탭으로 이동 → 필터: **pending**.
2. 해당 타일 **승인** 또는 **거부**(사유 입력).
3. 거부된 타일은 라벨러가 **라벨링** 탭에서 수정 후 재저장 → pending으로 다시 올라올 수 있음.

---

## 9. 자주 발생하는 문제

| 증상 | 원인 및 해결 |
|------|-------------|
| 타일 목록이 비거나 404 오류 | **데이터셋** 탭에서 올바른 데이터셋을 선택했는지, 타일 생성을 했는지 확인 |
| 저장 후에도 상태가 바뀌지 않음 | **타일 목록 새로고침** 클릭 |
| 마스크가 저장 전 상태로 보임 | **다시 불러오기** 버튼으로 서버 데이터와 동기화 |
| SAM 실행 후 마스크가 이상하거나 변화 없음 | 여러 자재에 양성을 한꺼번에 찍지 말고 **자재마다 양성 1점**으로 시도. Undo 후 브러시로 직접 수정하거나, **양성 1 + 음성**으로 같은 객체만 다듬기. |
| 단축키가 동작하지 않음 | 사이드바 입력 필드에서 클릭해 포커스를 빼고 시도 |
| Export 실패 | labeled 타일이 0개이면 실패; 타일을 1개 이상 저장한 후 재시도 |
