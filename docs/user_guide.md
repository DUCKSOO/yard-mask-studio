# yard-mask-studio 사용자 가이드 v1.0

**대상**: 라벨링 작업자·검수자. 서버 설치나 코드 수정 없이 브라우저만으로 작업하는 경우를 기준으로 한다.

> 라벨 판정 기준(occupied/non_occupied/ignore)은 [labeling_guide.md](./labeling_guide.md) 를 참고한다.  
> 서버 설치·환경 구성은 [README](../README.md) 를 참고한다.

---

## 1. 시작 전 확인사항

1. 팀에서 안내한 **브라우저 URL**로 접속한다 (예: `http://localhost:5173`).
2. 서버(백엔드)가 켜져 있어야 한다. 흰 화면이나 API 오류가 뜨면 담당자에게 문의한다.
3. **tenant ID**와 **dataset ID**를 미리 확인한다. 담당자가 사전에 공유한다.

## 3. 사이드바 항목별 설명

### 3-1. tenant / dataset

- **tenant**: 서버에 등록된 조직 단위 (보통 `default`).
- **dataset**: 작업할 특정 이미지 세트 ID. 잘못 입력하면 타일 목록이 비거나 404 오류가 난다.
- 입력 후 **타일 목록 새로고침** 버튼을 누른다.

### 3-2. 진행률 바

- 새로고침 직후 `전체 N | labeled M | approved P` 한 줄로 표시된다.
- 파란 막대: labeled 비율, 초록 막대: approved 비율.

### 3-3. TileNavigator (타일 그리드)

- 상단 탭으로 상태별 필터: **전체 / 미라벨 / labeled / approved / rejected**
- 각 버튼은 상태에 따라 색상이 다르다.

| 색상 | 상태 |
|------|------|
| 회색 | unlabeled (미라벨) |
| 파랑 | labeled |
| 초록 | approved |
| 빨강 | rejected |

- 버튼을 클릭하면 해당 타일 이미지와 기존 마스크를 캔버스에 불러온다.
- 타일 ID가 길 경우 뒷부분만 표시되며, 마우스를 올리면 전체 ID가 tooltip으로 보인다.

### 3-4. ClassPanel (클래스 선택)

브러시로 칠할 **클래스(색상)**를 고른다. 현재 프로젝트 기본값:

| 클래스 | ID | 의미 |
|--------|----|------|
| non_occupied | 0 | 야드 바닥·통로 |
| occupied | 1 | 컨테이너·장비·적재물 |
| ignore | 255 | 경계 불확실·가림 |

### 3-5. ToolBar (도구)

| 도구 | 설명 |
|------|------|
| 브러시(B) | 선택한 클래스로 마스크를 칠한다 |
| 지우개(E) | 해당 픽셀을 배경(0)으로 지운다 |
| 점(+) | SAM에 전경 프롬프트 점을 찍는다 |
| 점(−) | SAM에 배경 프롬프트 점을 찍는다 |
| 박스 | SAM에 바운딩 박스 프롬프트를 그린다 |
| 이동(P) | 캔버스를 드래그로 이동한다 |

**반경 슬라이더**: 브러시·지우개 도구가 선택된 경우에만 표시된다 (2 ~ 80px, 기본 10px).

**SAM 실행**: 프롬프트(점·박스)가 1개 이상 있을 때 활성화된다. 클릭 후 잠시 기다리면 마스크 후보가 적용된다. 결과가 마음에 들지 않으면 Undo 후 브러시로 직접 수정한다.

### 3-6. 액션 버튼

| 버튼 | 동작 |
|------|------|
| Undo | 직전 마스크 변경 취소 |
| Redo | 취소된 변경 다시 적용 |
| 저장 | 현재 마스크를 서버에 저장 (타일 상태 → `labeled`) |
| 다시 불러오기 | 서버에 저장된 마스크로 초기화 |

### 3-7. Export 패널

- **U-Net Export 실행** — 현재 dataset의 labeled 타일 전체를 묶어 export한다.
- 완료 후 **ZIP 다운로드** 링크가 나타난다.
- 오류 시 메시지와 **초기화** 버튼이 표시된다.
- Export는 마일스톤 시점에만 실행하는 것을 권장한다 (데이터가 충분히 쌓인 후).

### 3-8. 설정 영향도 위젯

- `tile_size`와 `tile_overlap`을 가정값으로 입력하고 **영향 분석**을 누르면, 현재 타일 수 대비 예상 변화량(Δ)을 보여준다.
- 실제 데이터는 변경되지 않는다 (미리보기 전용).
- 일반 라벨러는 사용하지 않아도 된다.

### 3-9. 검수(Review) 패널

- 패널 토글 버튼으로 펼칠 수 있다.
- 필터: pending / approved / rejected / all
- 각 항목에서 **승인** 또는 **거부**(사유 입력 가능)를 처리한다.
- 라벨러는 저장 후 검수 큐에 올라온 항목을 확인하고, 거부 사유를 참고해 해당 타일을 수정한다.

## 4. 캔버스 조작

| 조작 | 방법 |
|------|------|
| 확대·축소 | 마우스 **휠** |
| 캔버스 이동 | **휠 버튼** 드래그 또는 이동 도구(P) |
| 마스크 칠하기 | 브러시(B) 선택 후 **클릭·드래그** |
| 마스크 지우기 | 지우개(E) 선택 후 **클릭·드래그** |
| SAM 점 프롬프트 | 점(+)·점(−) 도구 선택 후 클릭 |
| SAM 박스 프롬프트 | 박스 도구 선택 후 드래그 |

**그리드 오버레이**: 야드 구획 기준 격자가 반투명으로 표시된다. 캔버스 확대 시 격자도 같이 확대된다.

## 5. 단축키 요약

| 단축키 | 동작 |
|--------|------|
| **Ctrl+Z** / **Cmd+Z** | Undo |
| **Ctrl+Y** / **Cmd+Y** | Redo |
| **Ctrl+S** / **Cmd+S** | 저장 |
| **B** | 브러시 도구로 전환 |
| **E** | 지우개 도구로 전환 |
| **P** | 이동(팬) 도구로 전환 |

> 텍스트 입력 필드(`tenant`, `dataset` 등)에 포커스가 있을 때는 단축키가 동작하지 않는다.

---

## 6. 일반적인 작업 순서 (라벨러)

1. **tenant / dataset** 입력 → **타일 목록 새로고침**
2. TileNavigator에서 **미라벨(회색)** 타일 클릭
3. ClassPanel에서 칠할 **클래스 선택**
4. (선택) SAM으로 후보 생성 → 브러시·지우개로 보정
5. **Ctrl+S** 또는 저장 버튼
6. 다음 타일 클릭 → 반복

---

## 7. 일반적인 작업 순서 (검수자)

1. Review 패널 열기 → 필터: **pending**
2. 해당 타일 확인 (필요 시 TileNavigator에서 직접 클릭해 캔버스로 확인)
3. **승인** 또는 **거부**(사유 입력 후)
4. 거부된 타일은 라벨러가 수정 후 재저장 → pending으로 다시 올라옴

---

## 8. 자주 발생하는 문제

| 증상 | 원인 및 해결 |
|------|-------------|
| 타일 목록이 비거나 404 오류 | `tenant`·`dataset` 입력값 확인, 서버 기동 상태 확인 |
| 저장 후에도 상태가 바뀌지 않음 | **타일 목록 새로고침** 클릭 |
| 마스크가 저장 전 상태로 보임 | **다시 불러오기** 버튼으로 서버 데이터와 동기화 |
| SAM 실행 후 마스크가 이상하거나 변화 없음 | Undo 후 브러시로 직접 수정; 프롬프트 점/박스 재배치 후 재시도 |
| 단축키가 동작하지 않음 | 사이드바 입력 필드에서 클릭해 포커스를 빼고 시도 |
| Export 실패 | labeled 타일이 0개이면 실패; 타일을 1개 이상 저장한 후 재시도 |
